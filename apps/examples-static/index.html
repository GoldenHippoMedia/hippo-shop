Hippo Shop SDK

Live demos of declarative product binding

Three real-world archetype pages — funnel step, offer selector, PDP — rendered from real Gundry MD / MCT Wellness data using only HTML data-* attributes. Each one mirrors how partners actually build on the SDK.

The demos

Three archetypes, all using MCT Wellness data

Open any file directly in a browser — no build step, no bundler. Each demo loads the SDK from api-uat.goldenhippo.io with a public test key.

Demo 01 · funnel resource

Funnel step

Single step of a multi-step funnel, with a progress bar built from funnel.steps[] and a "Continue to step N" CTA whose href is built from the next step's slug.

  • <template data-each="steps"> for the progress bar segments
  • data-attr-data-stage="kind" for per-step-kind CSS coloring
  • gh:bindings-ready listener that reads funnel.steps[currentIndex + 1]
  • Cached window.gh.data.funnel(slug) call from inline JS

Demo 02 · destination resource × 12

Offer selector

12 destinations on one page (2 flavors × 3 quantities × 2 purchase types). Filter pills collapse the 12 down to 3 visible cards. Each card's CTA routes to its post-purchase funnel.

  • 12 data-gh-destination resources fetched concurrently (deduped by the SDK)
  • data-attr-href="funnelSlug" on each card's CTA — no JS for routing
  • data-if="pricing.price.savings" ribbons on the value-tier cards
  • Pill-driven [hidden] toggling — no extra fetches when filters change

View demo

Demo 03 · product resource

PDP (Product Detail)

Single product with a tier toggle (Standard / My Account) and a quantity ladder. Tier swap is CSS-only — both tier card sets are pre-bound; no gh.refresh() required.

  • data-with="variants.subscription.standardByQuantity.<qty>" for scope narrowing per card
  • Per-attribute formatter override (data-attr-format-data-stock)
  • Custom savePercent formatter registered via gh.format.register
  • One product fetch covers six pre-bound variant cards

The SDK ships as @goldenhippo/hippo-shop-sdk — see the SDK README for installation, the data model, and the full attribute reference.

The publishable key embedded in each demo's <script> tag (gh_pk_test_all_…) is a public, scoped test/UAT key — it's intentionally checked in and safe to ship in client HTML.

Every demo is a single self-contained HTML file: open it directly in a browser, no build step required.
