Hippo Shop SDK

Live demos of declarative product binding

These four pages render production-quality cards using only HTML data attributes — no JavaScript per page. Each one focuses on a different slice of the SDK so you can compose pieces into your own design.

The demos

Four pages, four slices of the SDK

Open any file directly in a browser — no build step, no bundler. Each demo loads the SDK from api-uat.goldenhippo.io with a public test key.

Demo 01

Product pricing

A single, polished product card with the SDK's advanced features.

  • Custom formatter registration via window.gh.format.register
  • Three review fields including reviews.globalFiveStarReviews
  • data-attr-href, data-attr-aria-label, and data-attr-data-product-id on the CTA
  • Subscription cadence + "or one-time" comparison via alternatePurchaseTypePrice

View demo

Demo 02

Variant grid

Subscription, one-time, and My Account tiers from a single product fetch.

  • <template data-each="standardList"> for iteration
  • Direct lookup via standardByQuantity.<qty> for featured packages
  • bool formatter for in-stock pill, with paired data-attr-data-stock
  • data-if cascading so sections hide when their data is empty

View demo

Demo 03

Funnel steps

Render a funnel's step list with formatter range + programmatic refresh.

  • <template data-each="steps"> over the funnel
  • uppercase formatter for step kinds
  • data-attr-data-kind for CSS color-coding, with per-attribute format short-circuit
  • gh:data-ready listener + a Refresh button calling window.gh.refresh()

View demo

Demo 04

Destination offer selector

Six destinations on one page, with tabs, bump offers, and shipping.

  • Multiple data-gh-destination cards batched in one fetch
  • gh:data-ready event replaces 200ms CTA polling
  • <template data-each="pricing.bumpOffers"> for checkout add-ons
  • data-attr-href="funnelSlug" per row so the CTA is fully declarative

View demo

The SDK ships as @goldenhippo/hippo-shop-sdk — see the root README for installation, the data model, and the full attribute reference.

The publishable key embedded in each demo's <script> tag (gh_pk_test_all_…) is a public, scoped test/UAT key — it's intentionally checked in and safe to ship in client HTML.

Every demo is a single self-contained HTML file: open it directly in a browser, no build step required.
